## 批判プロトコル (各シナリオの Pro / Con / Unknowns)

### S0 baseline

**自動フラグ**: ⚪ 中立 (判定不能 / 大きな変化なし)

**擁護論 (Pro)**:
- 現状の挙動を参照する基準点として価値あり
- 変更による悪化リスクなし

**批判論 (Con)**:
- C で判明した H2 (+12.58pt 増幅) と H3 (B1 std 0.86x) の bias を抱えたまま
- mc3 系の根本問題が解消されない

**未検証論点 (Unknowns)**:
- このまま forward でどれだけ bias が累積するか

### S1 H2 単独 (linear)

**自動フラグ**: 🔴 赤 (baseline より明らかに悪化)

**擁護論 (Pro)**:
- 線形変換は単調かつ等比、高 prob 帯の異常増幅が原理的に生じない
- H2 が真の原因なら最も直接的な治療
- 1号艇 prob 60% が rating 上で他艇との不当な差を生むのを防ぐ

**批判論 (Con)**:
- compute_ratings 全体の挙動が変わるため、他の補正項 (展示タイム差/クラス係数等) との整合性が崩れる可能性
- scale=8.0 は heuristic、過去の logit base 設計の前提が崩れる
- 2 号艇・3 号艇の rating 順位が予期せず逆転する可能性
- 線形は確率の上下端でも線形なので、p=0.01 と p=0.99 の差が圧縮される

**未検証論点 (Unknowns)**:
- scale 値の感度 (8.0 vs 6.0 vs 10.0 で結果がどう変わるか)
- 他の std 補正項と組み合わせた時の最終 rating 分布形状
- forward 期間で勝率に乗るか

### S2 H3 単独 (A1 0.85)

**自動フラグ**: 🟢 緑 (1号艇 bias 改善 + 副作用小)

**擁護論 (Pro)**:
- 単一係数変更で済むため副作用が局所化
- A1 = 0.85 は他艇 (A2: 0.90) との差を縮め、設計の手触りが自然
- C の H3 が「A1 0.75 で std systematic に小さい」と示した直接の応答

**批判論 (Con)**:
- A1 0.75 は v10.3-10.6 開発時の集中チューニング結果で、外す根拠が薄い
- B1 std が大きくなると 1着率も低下 → 戦略全体の Kelly 計算に影響
- H3 が真の主因かは C では「部分該当 🟡」止まり、効果限定の可能性
- 1号艇以外の艇クラスでも A1 選手の挙動が変わる

**未検証論点 (Unknowns)**:
- 0.85 が最適か 0.80 か 0.90 か (S4 grid で別検証)
- 旧 v10 開発時に A1=0.75 を選んだ data 根拠が現存するか

### S3 H1+H2 (cal v2 + linear)

**自動フラグ**: 🔴 赤 (baseline より明らかに悪化)

**擁護論 (Pro)**:
- H1 (高確率帯 -4.41pt 過小評価) と H2 (logit +12.58pt 増幅) の相殺関係を両方絶つ
- calibrator v2 は 2 月-3 月 OOS で fit 済み、独立の補正レイヤー
- 線形化と組み合わせると bias の発生源が二重に消える

**批判論 (Con)**:
- 二箇所同時変更は副作用切り分け困難
- calibrator v2 は本番未投入 (Phase A A1 撤退済み)、forward 性能未検証
- H1 と H2 が相殺していたなら、両方直すと逆に過小評価に転じる可能性
- 線形化と calibrator の交互作用が予測しづらい

**未検証論点 (Unknowns)**:
- calibrator v2 単独 (S3a 相当) の効果
- 相殺バランスが崩れた時の系統的方向

### S4 A1=0.8

**自動フラグ**: 🟢 緑 (1号艇 bias 改善 + 副作用小)

**擁護論 (Pro)**:
- A1 係数の最適値を data で特定できる
- 単一係数の段階変動なので副作用が予測しやすい
- 0.80-0.95 のレンジは設計の自然な範囲

**批判論 (Con)**:
- grid search 自体は事後選択バイアスを生む (5828 races 上でフィットする値は本番でも最適とは限らない)
- 過去 data だけで最適化すると forward overfitting
- A1 のみ動かすので H2 logit 問題は残る

## 留意 (CLAUDE.md 批判プロトコル準拠)

- フラグ (緑/黄/赤) は heuristic 閾値判定であり、絶対基準ではない
- ROI proxy は top-3 単純購入で mc3 の Kelly/EV filter 不含、相対比較指標
- 全シナリオ (S0 含む) を同 seed (42) で再計算しているため、シナリオ差は純粋に rating/std/calibrator 由来
- 本 cache の元 QMC は別 seed で生成されているため、49_qmc_vs_empirical.py の数値と S0 は微差あり
- KL は対称性なし、QMC → actual 一方向のみ
- 結論は出さず、岩下さんの判断 (どのシナリオを B 本実装で採用するか) を待つ
