# 3連単 組み合わせ手法 研究 v1 — 設計書

- **日付**: 2026-07-14
- **状態**: 設計（実装プラン作成前）
- **対象**: boatrace-predictor / オッズ盤台帳(`race_odds_board`)＋marginals(`predictions`)＋結果(`races`)＋QMC結合エンジン(`src/monte_carlo.py`)
- **種別**: read-only 研究（本番・DB・運用設定は一切変更しない）

---

## 1. 背景（ここまでで分かっていること）

- 「戦略を増やすたびサンプルがゼロに戻り真理が見えない」ループを断つため、2026-06-30 に**オッズ盤台帳**を稼働。全120通りのオッズを毎レース保存し始めた（~160レース/日で蓄積、現在約2,183レース）。
- 台帳を使った初回全盤分析（2,183レース・261,960通り）で判明:
  - **市場の強い favorite-longshot bias**: 極端穴(200倍超)の blanket 回収 **46%**（激しく買われすぎ）／中低オッズ(5-40倍) **80-84%**。どの帯も100%未満＝盲目的全買いでは勝てない。**エッジは選別にある**。
  - **モデルの組み合わせ順位に実スキル**: モデル上位1点の回収 **97%**（全体平均57%比 +40pt）、上位から単調劣化。※これは超ざっくり近似(p1×p2×p3)＋無フィルタで既にほぼ損益分岐。
  - **モデルは"当てる"でなく"価値を拾う"**: モデル本命の的中率(8.7%)は市場本命(9.8%)よりわずかに低いが回収は上＝割に合う目を選んでいる。
- 各順位の周辺確率は較正良好（1着 55.9%=55.9%、2着 ECE 0.31%、3着 0.61%）。弱点はモデルの確率でなく**買い方・組み合わせ側**、が支持される。
- 既存 P/P2（得意場）は簡易版の組み合わせ選別（オッズ帯＋EV＋較正）で回収~140%まで来ている＝機構は既に効いている。

**結論済み**: 組み合わせ手法のエッジは「ある」。方向（極端穴カット×中オッズ集中×モデル上位選別×価値取り）も見えた。
**未検証（本設計のテーマ）**: それを**本物のQMC結合**で組み、**out-of-sample で本当にプラスに乗るか**。

## 2. 目的と検証したい問い

**問い**: 「市場の歪み × モデルの組み合わせ順位」を突く3連単の組み合わせ手法に、**言語化できる機構**と**out-of-sample のエッジ**があるか。

**成果**: どちらの結論も価値。
- あれば → 機構を明示し、前向き検証で確定させ、次段（ケリー/複利最適化）へ。
- なければ → 「市場が既に織り込んでいて寺銭を超えない」と明確に結論し、別アプローチへ。

## 3. 決定的な制約（データ）

- **全オッズ盤は 2026-06-30 以降のみ**。過去にオッズ盤が無く**バックフィル不可**。
- したがって「過去数ヶ月で train/test」はできない。**2週間分＝暫定の一次読み**、**これから貯めながらの前向き検証が本検証**。
- 器具は**増え続けるデータで繰り返し再実行**でき、初期窓と後続窓の縮み(shrinkage)を必ず併記する設計にする。

## 4. スコープ

**含む**:
- **本物QMC結合を核**とした joint 推定（`qmc`=本物QMC v3・v4 再計算が主。`proxy`=p1×p2×p3 は sanity 対照のみ）
- 市場モデル（de-vig 含み確率）
- 診断（joint較正・歪みマップ・モデルvs市場 差マップ）
- **本番さながらの高忠実度シミュレータ**（実ケリー・1点/レース上限・最低賭け金・100円丸め・**判断時オッズで賭け→結果配当で決済**を再現）
- **シミュレータ検証ゲート**：既存戦略（P/P2/V11）を同シミュレータで回して**DBの実績PnLを許容誤差内で再現**できることを、新手法評価の前提にする（再現できないシミュレータの数字は信用しない）
- **事前登録した1候補手法**の評価（フラット賭け＝素のエッジ／本番同等賭け＝実収益 の両方）
- 研究レポート `RESULTS.md`

**含まない（YAGNI / 将来段）**:
- ケリー**割合の最適化**（複利最大化のチューニングは次段。本番同等シムは既存の実ケリー設定を"再現"するのみで、割合を探索しない）
- shadow 本番並走・`betting_config`/`scheduler` 変更
- 過去バックフィル（不可）
- リアルタイムのライブ手法配信

## 6. アーキテクチャ（`analysis/combo/` パッケージ、小さな責務単位）

```
loader:   台帳⨝marginals⨝boats⨝races⨝結果 → レース単位の candidate 集合
joint:    qmc(本物再計算, 主) / proxy(p1×p2×p3, 対照) → race_id → {combo: model_prob}
market:   オッズ盤 → {combo: 市場含み確率}（de-vig）
diagnose: (a)joint較正 (b)歪みマップ (c)モデルvs市場 差マップ
method:   設定可能な手法関数(フィルタ→採点→選別→賭け金) → race_id → [(combo, stake)]
sim:      本番同等シミュレータ。手法の買い目を、実ケリー・1点上限(max_ticket)・
          レース上限(max_total)・最低賭け金・100円丸めで賭け、判断時オッズで建て、
          結果配当で決済 → equity/ROI/PnL/DD/的中/集中度
validate: 既存戦略(P/P2/V11)を sim で回し、DB実績PnLを許容誤差内で再現できるか検証
          （通らなければ sim を直す。ここが緑になって初めて新手法の数字を採用）
evaluate: 検証済み sim で新手法をリプレイ。フラット賭け(素のエッジ)と本番同等(実収益)
          の両方、任意日で train/test（前向き窓）分割
report:   全実行 → RESULTS.md（診断＋sim検証＋候補手法＋判定＋前向き追跡の指示）
```

各モジュールに assert ベースの sanity テスト（既知値の再現）。joint / market / method / sim は**差し替え可能なインターフェース**にして独立にテスト・比較できるようにする。**sim は「既存戦略のDB実績を再現できる」ことをもって忠実度を担保**する（＝本番さながらの根拠）。

## 7. 事前登録する候補手法（v1）— 測る前に確定・commit

**機構（すべて言語化）**:
1. **オッズ帯フィルタ**: 5 ≤ odds ≤ 40（市場が最もマシな帯。回収46%の200倍超・薄い1-5倍を除外）。
2. **採点**: 各目を `edge = model_prob − market_implied`（モデル結合が市場含みを上回る度合い）。
3. **選別**: `edge > 0` の上位 **3点**（k=3, 閾値0 を事前登録）。
4. **賭け金の見せ方**: (a)フラット（各100円）＝**素のエッジ**判定用、(b)本番同等（既存の実ケリー設定を sim で再現）＝**実収益**判定用。両方を報告。ケリー**割合の探索・最適化はしない**（次段）。
5. **joint**: 主判定は `qmc`（本物）、対照に `proxy` も併記。
6. **前提**: §6 の sim 検証ゲート（既存戦略のDB実績再現）が緑であること。緑でなければ本番同等の数字は出さない。

**合格ライン（事前登録・暫定窓＋前向き）**:
- ROI が明確に 100% 超（寺銭を超える）
- **広く勝つ**（最大1本が総回収の一定割合未満＝単発大穴依存でない）
- **市場基準（全帯 blanket／市場本命買い）を上回る**
- 上記が**初期窓と後続の前向き窓の両方で方向一致**（縮みが許容範囲）

判定は出すが**採否は岩下さん**。「明らかな却下（ROI<市場基準 等）」は明示。

## 8. 規律（過学習対策＝本研究の生命線）

- 候補手法の仕様を**測定前に git commit**（後から良い設定を探して有意を作らない）。
- **2週間＝一次読み、前向き蓄積で本検証**。器具は増えるデータで再実行し、**初期窓 vs 後続窓の縮みを必ず併記**。
- **言語化できる機構のみ**（各部品に物理/市場構造の説明）。説明できない黒箱は不採用。
- **集中度を常に報告**（大穴一発の見かけを暴く）。
- 変種を試すときは**別の事前登録 run**として記録（多重検定を意識、こっそり最良を選ばない）。
- boatrace CLAUDE.md の検証プロトコル（標本数・PnLとROI併記・市場織り込み・QMC重複）を診断・判定に適用。

## 10. 前提・リスク

- **標本が薄い**（全盤2週間・的中は少数）。ROI判定は前向きで確度を上げる前提。市場マップ(26万通り)は堅牢だがモデル手法の成績は初期窓の一次読み。
- **QMC再計算の入力**（boats_data の展示等）が欠けるレースは除外し、除外率を明記。
- **「エッジ無し」という否定的結論もあり得る**。その場合も明確に結論する。
- 季節性（春〜夏に偏る）。結論は同季節への一般化に留める。
- 本物QMCと簡易近似で結論が食い違う場合、**本物QMCを主**とし差分を明示。
- **シミュレータ自体の誤りリスク**：sim が間違っていれば全結論が崩れる。だから
  **既存戦略のDB実績再現を通過ゲートにする**（再現できないうちは新手法の数字を出さない）。
  再現誤差の許容幅は実装プランで定義（例：主要戦略の期間PnLが±数%以内）。
- **本番同等（ケリー）は変動を増幅**：薄い2週間窓ではケリー収益が上振れ/下振れしやすい。
  だから**素のエッジ（フラット）を一次判定の主**とし、本番同等は"実際いくらか"の参考。
  ケリー割合の最適化は次段（エッジ確定後）。
