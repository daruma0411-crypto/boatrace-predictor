# ボートレース予想AI: EVゾーン集中戦略の変遷

## システム概要

- **モデル**: PyTorch 208次元特徴量、4モデルアンサンブル (27,826レースから学習)
- **予測対象**: 3連単 (1-2-3着の順番を当てる、全120通り)
- **オッズ取得**: boatrace.jp から全120通りのリアルオッズをスクレイピング
- **EV計算**: `EV = モデル確率 × 割引オッズ` (割引: 低オッズ0.85、中0.88、高0.95)
- **ベット額**: Kelly基準で算出 (実質ほぼ全て100円均一)
- **運用**: Railway上でStreamlit + PostgreSQL、スケジューラが自動ベット

---

## Phase 1: 初期運用 (3/11-3/14)

### 設定
- 6戦略並列A/Bテスト (conservative, standard, divergence, high_confidence, ensemble, div_confidence)
- min_EV = 0.0 (フィルタなし)、max_EV = なし
- Kelly基準でベット額算出（実質100円均一）
- 初期資金: ¥200,000/戦略

### 結果
- **全676件中 14的中 (2.1%)**
- **投資: ¥172,800 → 回収: ¥79,960 → ROI 46%**
- モデル平均確率 3.6% vs 実的中率 2.1% → **2〜4倍の過大推定**

### 問題
- モデルが確率を系統的に過大推定
- 全戦略マイナスROI

## Phase 2: キャリブレーション試行 (3/15 午前)

### やったこと
1. `calibration_analysis.py` で確率帯別の過大推定率を定量化
2. `backtest_with_real_odds.py` でグリッドサーチ (cal_factor × min_ev × max_bets)
3. キャリブレーション係数を `betting.py` に組み込み

### 結果

- 最良: cal_factor=0.9, min_ev=1.1, max_bets=3 → ROI 46.78%
- min_EV引き上げ → ベット件数激減、効果不明

## Phase 3: EV逆転現象の発見 (3/15 午後)

### きっかけ
3/15当日のCSVエクスポート (2,251件) を詳細分析。

### 衝撃の発見: EVと的中率が逆相関

| EV帯 | 件数 | 的中 | 的中率 | ROI |
|------|------|------|--------|-----|
| **0.5-0.8** | 584件 | 43本 | 7.4% | **177%** |
| 0.8-1.0 | 491件 | 21本 | 4.3% | 91% |
| 1.0-1.2 | 569件 | 17本 | 3.0% | 81% |
| 1.2+ | 499件 | 4本 | 0.8% | 45% |

**高EVほど的中しない。低EVほど的中する。**

### なぜEVが逆転するのか？

```
EV = モデル確率 × 割引オッズ

高EV = モデルが「市場より大幅に有利」と判断
     = モデルと市場オッズの乖離が大きい
     = モデルが過大推定している可能性が高い

低EV = モデルが「市場とほぼ同意見」
     = モデルの推定が控えめ
     = 確率推定が最も正確なゾーン
```

市場オッズは数百万人の集合知で概ね正確。モデルが市場と大きく乖離するほど、モデル側が間違っている。

これは統計学の **calibration paradox** に該当: 予測モデルは「自信がない時ほど正確」。

## Phase 5: v8 EVゾーン集中戦略 (3/15 夜〜3/16 現在)

### 設計思想
- **高EVを排除し、低EVゾーンに集中する**
- 「市場と大きく乖離するベットほど負ける」というデータに基づく
- オッズによるフィルタリングは「結果でフィルタリング」になるため危険 → EVのみで判断

### 設定 (v8)

| 戦略 | Kelly | EV範囲 | フィルタ | 役割 |
|------|-------|--------|---------|------|
| high_confidence | 1/5 | 0.5-0.8 | entropy H<2.3 | エース |
| standard | 1/4 | 0.5-0.8 | なし | カバー |
| conservative | 1/8 | 0.5-1.0 | なし | 安全策 |

- 不採用: optuna, divergence, div_confidence, ensemble, bt_*系 (全て低ROI)
- 日次ベット上限: 30件/戦略
- Kelly負 (EV < 1.0) の場合: 最低額100円でフラットベット

### 3/16 途中経過

- ベット: 91件 (3戦略 × 約30件)
- 1号艇1着: 83件 (91%) ← 3/15の81%と概ね同じ (誤差範囲)
- 投資: ¥9,100
- 確定結果: まだ少数

## 未解決の論点

### 1. サンプルサイズの問題
- EV 0.5-0.8 = ROI 177% は **3/15の1日データのみ**
- 3日間 (3/16-18) の検証を予定

### 2. 1号艇偏重
- EV 0.5-0.8は構造的に1号艇軸に偏る (モデルと市場が合意しやすい)
- 1号艇ROI 117% (薄利) vs 2-3号艇ROI 262-1022% (大利)
- 利益の源泉は少数の2-3号艇的中 → 運の要素が大きい可能性

### 3. Kelly基準が機能しない問題
- EV < 1.0 → Kelly値が負 → 数学的に「賭けるな」の領域
- しかし実績ではROI 177% → Kelly基準の前提 (正確な確率推定) が成立していない
- 現在は100円フラットベットで運用

### 4. モデル確率の根本的課題
- モデルは確率を2-4倍過大推定
- 過大推定が「均一」でないため、EVゾーンフィルタが有効に機能
- モデル再学習 (追加10,000レースのデータ収集済み) で改善の余地あり

## 議論したいこと

1. **EV 0.5-0.8ゾーンへの集中は統計的に妥当か？** 1日のデータで戦略を決めるリスク。
2. **Kelly基準を無視して100円フラットベットを続けるのは合理的か？** EV < 1.0でベットする数学的正当性。
3. **モデル改善とEVゾーン戦略、どちらを優先すべきか？** モデルの確率推定が改善されれば、EVゾーンの有効性も変わる可能性。
4. **他に試すべきアプローチは？** (例: 市場オッズベースのモデル補正、Isotonicキャリブレーション、etc.)
